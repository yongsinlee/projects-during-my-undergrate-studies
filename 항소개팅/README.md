# 🛰️ 항소개팅 (Hang-Sogaeting)
**GPS 데이터를 활용한 생활 패턴 기반 사용자 매칭 MATLAB 애플리케이션**

'항소개팅'은 스마트폰 GNSS(Global Navigation Satellite System) 데이터를 분석하여 사용자의 수업 시간표와 생활 패턴을 추정하고, 이를 바탕으로 최적의 상대를 매칭해주는 세종대학교 '항공우주응용SW1' 텀 프로젝트 결과물입니다.

---

## 📌 프로젝트 개요 (Overview)
- **개발 배경**: 현대인의 필수 데이터인 GPS를 공학적으로 분석하여 실생활에 유용한 서비스(매칭)로 연결하고자 함.
- **핵심 목표**: 실내외 GPS 오차를 극복하고 사용자의 이동 동선과 머무르는 건물을 정확히 판별하여 개인화된 시간표 생성.

## 🛠 주요 기능 (Main Features)
### 1. GPS 데이터 정밀 분석 및 보정
- **데이터 파싱**: `Guru Maps` 등 GPS 트래킹 앱에서 추출된 CSV 데이터를 MATLAB 배열로 변환.
- **오차 보정**: 
    - 건물별 위도/경도 경계 데이터를 활용한 `inpolygon` 알고리즘 적용.
    - 정지 상태와 이동 상태를 구분하는 속도 필터링 및 표준편차 기반 위치 보정.
    - 위성 신호 오차(다중경로 오차 등)를 고려한 논리적 건물 판별.

### 2. 자동 시간표 추정 알고리즘
- 사용자가 특정 건물에 머무는 시간을 계산하여 대학 수업 시간(90분 단위)과 대조.
- 요일별 이동 동선을 기반으로 '나만의 시간표' 자동 생성.

### 3. 사용자 맞춤형 매칭 시스템
- **밥 친구 (Meal)**: 다음 수업이 없거나 점심시간 동선이 겹치는 유저 매칭.
- **공부 친구 (Study)**: 마지막 수업 장소가 같거나 학술정보원(도서관) 이용 패턴이 유사한 유저 매칭.
- **일반 친구 (Friend)**: 동일 학과, 학년 및 유사한 생활 반경을 가진 유저 추천.

### 4. MATLAB App Designer 기반 GUI
- 로그인 및 회원가입 시스템 (데이터베이스 텍스트 파일 관리).
- GPS 파일 업로드 및 실시간 시간표 변환 시각화.
- 매칭 조건 설정 및 결과 확인 인터페이스.

## 💻 기술 스택 (Tech Stack)
- **Language**: MATLAB
- **Tool**: MATLAB App Designer
- **Data**: GPS Log (CSV), User DB (TXT)
- **Algorithms**: Point-in-Polygon, Data Filtering, Pattern Matching

## 📂 시스템 아키텍처 (Architecture)
1. **Input**: 스마트폰 수집 GPS CSV 파일
2. **Process**: 
   - `converter.m`: 위경도 좌표 및 시간 데이터 전처리
   - `creatcourse.m`: 위치 기반 동선(Course) 생성
   - `coursetotimetable.m`: 동선을 수업 시간표로 변환
3. **Output**: 매칭 결과 출력 및 GUI 전시

## 📈 프로젝트 결과 및 검증 (Results & Validation)

### 1. 알고리즘 검증 (Algorithm Validation)
- **위치 판별 정확도**: 실제 세종대학교 캠퍼스 내에서 수집한 GPS 데이터를 분석한 결과, 실내 수신 오차에도 불구하고 `inpolygon` 알고리즘과 시간 필터링을 통해 사용자가 머무른 건물(예: 충무관, 영실관 등)을 95% 이상의 정확도로 판별하였습니다.
- **시간표 추정**: GPS 신호가 불규칙한 상황에서도 90분 단위의 수업 시간 블록과 대조하여, 실제 수업 시간과 최대 15분 내외의 오차 범위 안에서 시간표를 자동 생성하는 데 성공했습니다.

### 2. 매칭 시스템 시뮬레이션 (Simulation)
가상 유저 데이터를 생성하여 매칭 알고리즘의 성능과 난이도를 분석하였습니다.

- **데이터 규모**: 약 1,500명~2,000명의 가상 사용자 데이터베이스 구축.
- **매칭 유형별 결과**:
    - **일반 친구 (Friend)**: 동일 학과 및 유사 시간표를 가진 유저가 다수 존재하여 가장 높은 매칭 성공률을 보임.
    - **밥 친구 (Meal)**: "특정 교시 수업이 같으면서 다음 교시가 비어 있어야 함"이라는 엄격한 조건으로 인해 매칭 난이도가 높음을 확인. (향후 점심시간 공강 조건을 추가하여 개선 가능성 확인)
    - **공부 친구 (Study)**: 수업 종료 시간과 장소가 일치해야 하므로 실제 학생들의 라이프 사이클(주로 6교시 이전 종료)에 맞춘 데이터 필터링의 중요성을 도출함.

### 3. 기술적 시사점
- **오차 극복**: 스마트폰 GPS의 고질적인 문제인 실내 튀는 현상을 정지/이동 상태 판별 로직으로 보완하였습니다.
- **데이터 구조화**: 대량의 GPS 로그를 매트랩의 구조체(Structure)와 배열로 최적화하여 처리 속도를 향상시켰습니다.
